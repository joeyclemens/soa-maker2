The Layout

The SoA that you have just exported is based on the Areas/Departments/Sub-Departments/Rooms you selected in the SoA Maker. You will be presented with a Summary page and a tab named SoA that contains the room list.

The formulas and formatting should be applied dynamically so there shouldnt be too much to do other than any specific things you want to add in or change.

The Summary tab

The Summary tab is where you can see a summary of the Departments and Sub Departments that you selected.

  • The Net Departmental Area column is a sub total of each Sub Department from the other tab. It calculates this using a SUMIFS formula to get the information as quickly and accurately as possible.
  • Circ % is data pulled from the database. This can be changed here, or changed in the database if a permanant change is needed. The data is taken from the Template SoA used to create the database.
  • Circ Area (m2) is calculated using the formula =SUM((D2+G2)*F2).1
  • Planning allowance Total (m2) is calculated using the formula =H2*D2
  • Plan % is set to 5% on all rows. This can be changed if you have a specific allowance for a particular Sub Department and the rest of the formulas will adjust themselves accordingly.
  • Engineering allowance (%) is set to 5% on all rows. This can be changed if you have a specific allowance for a particular Sub Department and the rest of the formulas will adjust themselves accordingly.
  • Engineering allowance Total (m2) is calculated using the formula =SUM((D2+G2)*H2)

1This and other formulas shown here are just examples. The actual formula will remain the same but will use different cells for reference

The SOA Tab

This tab will have the following columns:
AL3, AL2, AL1, Room_ID, Room_Description, Qty, Area, Total_Area, Comments, Room_ID, Room_Code2.

  • AL3, AL2, AL1 represent Area, Department and Sub Department from the SoA_Input_Form. The AL3 and AL2 columns are used in creating the Summary tab.
  • Room_ID is the unique identifier for each room as set in the database3.
  • Room_Description is the name of the room.
  • The Qty data has been taken from the Template SoA4. It can be changed and all formulas will adjust themselves accordingly.
  • Area is the total area of the room as set in the database. The information has come from the Template SoA4.
  • Total Area is the total area of the room multiplied by the Qty.
  • Comments have come from the Template SoA and are in the database4.

    • 2Room_ID & Room_Code are optional extras. They are for a quicker transfer to a database but should be ommitted if presenting to a client.

    3This data can easily be changed in the database should any of it be consistently wrong or just in need of updating.

    4See Reference Template SoA button at the top

Notable information

Formatting is created for each tab as the worksheet is created. The formulas are created dynamically based on the information passed on from the SoA Maker. The data should be accurate but its always wise to check the numbers as the script isnt clever enough to determine errors, let alone fix them as it goes.

This is still a work in progress and there are going to be bugs cropping up from time to time. See the Bugs and future revisions log for more information
